# ТЕХНИЧЕСКОЕ ЗАДАНИЕ НА РАЗРАБОТКУ РАСШИРЕНИЯ ДЛЯ БРАУЗЕРА

**Рабочее название проекта:** `[OwnSpace]`
**Платформы:** Firefox (приоритет), с резервированием под Chrome (кросс-браузерная архитектура).

## 1. Общее описание

Разработать расширение для браузера, полностью заменяющее стандартную страницу новой вкладки (`about:newtab`). Расширение является локальным аналогом SaaS-сервиса start.me. 
**Ключевое отличие:** отсутствие облачного бэкенда. Все данные хранятся локально в браузере пользователя. Обеспечивается максимальная приватность, минимальное потребление ресурсов и полная автономность работы (кроме явных запросов пользователя, таких как получение погоды или синхронизация CalDAV).

## 2. Техническая архитектура и стек

* **Манифест:** Manifest V3.
* **API:** Использование `browser.*` API с обязательным применением `webextension-polyfill` для обеспечения кросс-браузерности (Firefox + Chrome).
* **UI Framework:** Допускается использование легковесного фреймворка (Preact, Svelte или Vue 3 в урезанной сборке) для упрощения работы со состоянием и DOM. Чистый JS (Vanilla) также допустим, но фреймворк предпочтительнее из-за сложной логики DnD и виджета календаря.
* **Хранилище:** `browser.storage.local` (основное хранилище данных). Лимит в Firefox — ~10 МБ. Код должен быть оптимизирован для минимизации объема сохраняемых данных.
* **Криптография:** Встроенный `Web Crypto API` (алгоритм AES-GCM) для шифрования паролей CalDAV и опционального шифрования файла экспорта.

## 3. Функциональные требования

### 3.1. Система Workspaces (Рабочих пространств)

* **Навигация:** Вкладки располагаются в самом верху экрана.
* **Лимиты:** Максимум 10 рабочих пространств.
* **Управление:** 
  * Создание новой вкладки посредством клика на кнопку «+» справа от существующих (аналогично вкладкам браузера).
  * Переименование по двойному клику на названии вкладки.
  * Удаление (предусмотреть механизм удаления, например, через контекстное меню правой кнопкой мыши или крестик при наведении, чтобы избежать случайного удаления).
* **Изоляция:** Каждое workspace имеет свой уникальный набор виджетов, их расположение и задний фон.

### 3.2. Система виджетов и сетка

* **Сетка Layout:** Жестко фиксировано 3 колонки. Ширина любого виджета равна 1 колонке (виджеты не могут объединять колонки по горизонтали).
* **Высота:** Динамическая, зависит от контента внутри виджета.
* **Заголовки:** У каждого виджета есть заголовок, который пользователь может редактировать.
* **Сортировка (Drag-and-Drop):** По умолчанию виджеты выстраиваются сверху-вниз (Top-Down). Пользователь может перетаскивать виджеты для изменения порядка как по вертикали внутри одной колонки, так и переносить их в другие колонки.
* **Добавление виджета:** На пустом workspace в центре отображается призыв нажать «+» для добавления. При нажатии появляется выпадающее меню (dropdown) со списком доступных для добавления виджетов.
* **Удаление виджета:** Кнопка удаления (крестик или значок корзины) в шапке виджета.

### 3.3. Состав виджетов (MVP)

1. **Закладки (Bookmarks):** Локальное хранилище ссылок. Возможность создавать папки, добавлять URL, название ссылки. (Не использовать `browser.bookmarks` API, данные живут только внутри расширения).
2. **Заметки (Notes):** Простое текстовое поле для ввода и сохранения текста.
3. **Дата и Время:** Отображение текущих даты и времени. Обновление в реальном времени.
4. **Погода (Weather):** 
   * Интеграция с OpenWeather API.
   * Пользователь самостоятельно генерирует и вводит API-ключ в настройках виджета.
   * Если ключ не введен — виджет показывает плейсхолдер с просьбой получить ключ и ссылкой на сайт OpenWeather.
5. **Календарь (Calendar):** *Самый сложный виджет.*
   * Отображает сетку текущего месяца.
   * При клике на дату открывается форма добавления события (Поля: Название, Время начала, Время окончания). По умолчанию длительность — 1 час.
   * Визуализация: В ячейке даты события отображаются в виде цветных линий/полосок для наглядности загруженности дня.
   * Список событий: Внизу виджета выводится список событий для выбранной (активной) даты.
   * Редактирование: При клике на событие в списке открывается форма редактирования названия и времени.
   * **CalDAV Синхронизация (Опционально для пользователя):** Возможность подключить внешний CalDAV сервер (Nextcloud, Яндекс и т.д.).

### 3.4. Внешний вид (Кастомизация)

* **Темы:** Обязательна поддержка 2-х тем: Dark (темная) и Light (светлая). Переключатель тем должен быть доступен в интерфейсе.
* **Фон Workspace:** Настраивается индивидуально для каждого рабочего пространства. Доступные варианты:
  1. Сплошной цвет (color picker).
  2. Цветовой градиент.
  3. Изображение по URL-ссылке.
  4. Загрузка изображения с ПК пользователя.
     * *Ограничение:* Максимальный размер загружаемого файла — 2 МБ. Если пользователь выбирает файл больше — показать уведомление об ошибке/превышении лимита.
     * *Оптимизация:* Если файл <= 2 МБ, но большой (например, 1.9 МБ), расширение должно пропустить его через клиентское сжатие (например, через Canvas API) перед сохранением в `storage.local`, чтобы минимизировать расход лимита хранилища без видимой потери качества.

### 3.5. Экспорт / Импорт настроек

Так как облачного бэкенда нет, это единственный способ переноса данных.

* **Формат:** JSON-файл.
* **Логика Импорта:** Полная перезапись текущих данных расширения данными из файла (создание идентичного рабочего пространства).
* **Шифрование (Опционально):** При экспорте пользователь может установить пароль. Если пароль установлен, файл шифруется с помощью `Web Crypto API` (AES-GCM). При импорте зашифрованного файла система просит ввести пароль для расшифровки. Если пароль не был установлен при экспорте — файл сохраняется в открытом виде (plain text JSON).

## 4. Нефункциональные требования и Безопасность

* **Производительность:** Страница новой вкладки должна открываться мгновенно. Запросы к API (погода, CalDAV) должны выполняться асинхронно и не блокировать рендер интерфейса. Фоновые скрипты (Service Workers) не должны потреблять ресурсы, когда страница новой вкладки закрыта.
* **Безопасность (CalDAV):** Логин и пароль от CalDAV сервера **категорически запрещено** хранить в открытом виде в `storage.local`. Они должны шифроваться через `Web Crypto API`. Пользователю необходимо будет задать "Мастер-пароль" для расширения, которым будет шифроваться эта связка.
* **Офлайн-режим:** При отсутствии интернета виджет погоды скрывает данные/показывает ошибку сети. Календарь продолжит работать, отображая локально сохраненные события (написать "Нет связи" в зоне синхронизации).
* **Код:** Код должен быть чистым, модульным, без обфускации (требование магазина Firefox Add-ons - AMO).

## 5. Пользовательский путь (Onboarding)

1. Пользователь устанавливает расширение и открывает новую вкладку.
2. Он видит Demo-workspace (названный, например, "Добро пожаловать").
3. В Demo-workspace уже добавлены виджеты: Дата/Время, Поиск (если предусмотрен), Закладки (с 2-3 примерами ссылок), Погода (с просьбой ввести API ключ).
4. Пользователь может начать использовать Demo или нажать «+» для создания своего пустого workspace.
5. В пустом workspace пользователь видит призыв нажать «+», чтобы добавить первый виджет.

## 6. Требования к публикации (AMO / Chrome Web Store)

* Подготовить иконки расширения: 16x16, 32x32, 48x48, 96x96, 128x128.
* Иконки должны быть выполнены в едином стиле с интерфейсом расширения.
* Код не должен содержать минифицированных сторонних библиотек (только исходный код или ссылки на CDN из белого списка Mozilla).
